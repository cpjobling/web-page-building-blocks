  • Links which start with "/" are called absolute paths.
  • / is called a “virtual root” because it will not usually be routed at / in the file system.
  • Unix “forward slash” notation is used for directories on the web, even if another notation is used for files on the server.

To link to this page from anywhere on this website you could use HTML markup like this:

Go to the <a href="/about/index.html">About Us</a> page!.

Absolute paths

  • This is the About Us page. Its URL path is: public_html/eg-146/about/index.html.
  • Its URL is http://eg-146.cpjobling.org.uk/about/index.html
  • or more simply http://eg-146.cpjobling.org.uk/about/.
  • From the previous page we used the absolute path /about/index.html to get here.

Relative paths

  • To link to this document from youarehere.html you could use this HTML markup: 
    Go to the <a href="about/index.html">About Us</a> page!.

Linking upwards

To link to a file in a higher level directory you use the Unix notation for the parent directory. For example, from this file, ../ indicates the directory above ../public/eg-146/about, that is ../public/eg-146. This is the document root / of this web site.

  • To link to youarehere.html from this file you could use this HTML markup: 
    Back to <a href="about/index.html">You Are Here</a>!.
    Like this: Back to You Are Here!.

Images

The same rules applies to images. E.g. from this file the following are equivalent: 

<img src="../images/cpjobling.jpg" alt="The author of this web site." />
<img src="/images/cpjobling.jpg" alt="The author of this web site." />
<img src="http://eg-146.cpjobling.org.uk/images/cpjobling.jpg" 
     alt="The author of this web site." />
They all produce:
