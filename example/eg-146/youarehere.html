To link to this web page from anywhere on the Internet you could use HTML markup like this:

Go to <a href="http://eg-146.cpjobling.org.uk/youarehere.html">You Are Here!</a>.

Absolute paths

Can always use the absolute path to access a document.

  • This is the You Are Here page accessed by the full URL
    http://eg-146.cpjobling.org.uk/youarehere.html.
  • Its filesystem (Unix) path is: ../public_html/eg-146/youarehere.html which maps onto the web as /youarehere.html.
  • You can also access this page from anywhere on the example web site by using the absolute path: /youarehere.html.

Next [/about/index.html] →

Relative Paths

Relative links do not start with "/". The browser starts looking from the current directory, not the document root. To get to this page from the site home page you could use HTML markup like this:

Go to <a href="youarehere.html">You Are Here!</a>.

You can also reference documents relative to the current location.

  • From here you can link to a child page simply by using the folder name followed by the file name: about/index.html.
  • To link to a grand-child page simply by using the folder names followed by the file name:
    about/staff/index.html.

Linking offsite

You need the protocol and host to link to a resource offsite. For example this HTML markup would be used to link to another site:

Link to <a href="http://www.cpjobling.me/dokuwiki/eg-146:lecture14">lecture notes</a>.
